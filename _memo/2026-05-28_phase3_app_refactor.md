**ブランチ:** `feature/responses-api-refactor`  
**実施日:** 2026-05-28

---

## 改修概要

Chat Completions API + 単一チャットモードの旧実装を、Responses API + 2モード構成に全面書き直し。

---

## 変更点一覧

### 1. モデル変更

```python
# Before
model="gpt-4o-mini"

# After
model="gpt-5.4-mini"   # 定数 MODEL で管理
```

### 2. API 変更: Chat Completions → Responses API

| 項目 | 旧実装 | 新実装 |
|------|--------|--------|
| APIメソッド | `client.chat.completions.create` | `client.responses.create` |
| 会話履歴 | 全メッセージを毎回送信 | `previous_response_id` で参照するだけ |
| レスポンス取得 | `response.choices[0].message.content` | `response.output_text` |
| コンテキスト超過 | 自前管理なし | `truncation="auto"` で自動処理 |

### 3. 2モード構成

サイドバーのラジオボタンでモード切替。モード切替時はセッション状態を全リセット。

```
チャットモード  ← previous_response_id で会話継続
翻訳モード     ← 毎回独立リクエスト・対訳2列表示
```

### 4. セッション状態の再設計

| キー | 用途 |
|------|------|
| `current_mode` | 現在のモード（切替検出に使用） |
| `last_response_id` | チャットモード用・直前の応答ID |
| `chat_display` | チャット表示用ログ（APIには送らない） |
| `translation_display` | 翻訳結果の表示ログ |
| `conversation_count` | チャットモードの会話回数カウント |

### 5. チャットモードの制限値

| 項目 | 旧実装 | 新実装 |
|------|--------|--------|
| 会話上限 | 10回 | 10回（維持） |
| 入力文字制限 | 20文字 | **30文字**（プラン通りに修正） |

## システムプロンプト

### チャットモード（`CHAT_PROMPT`）

`_memo/プロンプトメモ.txt` に記録されていた設計をそのまま採用。

**ペルソナ:** 新潟県妙高市に生まれ育った70歳前後の女性  
**トピック制限:** 妙高市・上越地方・新潟県に関連する話題のみ対応。範囲外は短い一言で返す。  
**方言制限:** 辞書に載っている語彙・表現のみ使用。未掲載の方言・語尾は使わない。  
**辞書:** `dialect_dict.txt` の内容を `{dialect_dict}` で展開（全文埋め込み）

フェーズ2.5の実験結果（パターンA採用）に基づき、標準語→方言の2段階変換ではなく、最初のトークンから方言分布でデコードさせる直接生成方式。

### 翻訳モード（`TRANSLATION_PROMPT`）

翻訳専用のシンプルなプロンプト。辞書に基づいて標準語→妙高弁へ変換し、翻訳結果のみ出力。

## トークン節約の実装

| 施策 | 対象モード | 実装 |
|------|-----------|------|
| `previous_response_id` で会話継続 | チャット | 2回目以降は履歴トークンが増えない |
| `truncation="auto"` | チャット | コンテキスト超過を自動処理 |
| 毎回独立リクエスト | 翻訳 | 前の翻訳履歴を送らない |
| `max_output_tokens=200` | 翻訳 | 応答が不必要に長くなるのを防ぐ |
