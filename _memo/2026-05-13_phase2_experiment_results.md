## 実験概要

### 目的

辞書書式変更（フラット形式 → 3セクション構成）による翻訳品質・トークン数への影響を定量評価する。

### 条件

| 項目 | 設定 |
|------|------|
| モデル | `gpt-5.4-mini` |
| API | Responses API（`client.responses.create`） |
| テストフレーズ | 10文（`plans/experiment_phrases.txt`） |
| リクエスト方式 | 独立リクエスト（`previous_response_id` なし） |
| `max_output_tokens` | 200 |
| 実行回数 | 2回（ノイズ確認のため） |

### 辞書の違い

| | 旧辞書（`dialect_dict_old.txt`） | 新辞書（`dialect_dict.txt`） |
|---|---|---|
| 行数 | 180行 | 253行 |
| 構造 | フラット（アイウエオ順） | 3セクション・10カテゴリ |
| セクション | なし | `## 文法・語法ルール` / `## 語彙（品詞別）` / `## 会話例` |

## 考察

### トークン数

新辞書は旧辞書より約 **+1,400トークン/回** 増加する。これは辞書が73行増えた（180→253行）ことによる instructions の増量が原因であり、翻訳品質向上のためのトレードオフとして許容範囲。

キャッシュ率はRun2で旧辞書 93.7%・新辞書 90.2%と安定しており、定常運用では高いキャッシュヒットが期待できる。Run1のキャッシュ率が低かったのは初回実行でキャッシュが積まれる前だったため。

### 翻訳品質

**新辞書の改善点（2回とも再現）:**
- `すけ`（〜だから）の適用が安定（#4, #6）
- `こねだ`（この間）・`ごっつぉ`（ごちそう）・`えかった`・`たいそ` などの語彙適用が一貫している
- `いいわねや`・`そりゃえかったねや` など自然な文頭表現

**新辞書の問題点（2回とも再現）:**

| 問題 | Run1 | Run2 | 原因 |
|------|------|------|------|
| `】【` ノイズ混入 | #10 | #1 | モデル生成ノイズ。辞書中の `】（）` 記法（会話例のカッコ）が影響している可能性あり |
| `よに入って`（温泉が消える） | #9 | #9 | 辞書98行目の `湯 → よ` が「温泉」に誤適用されている |

### 対応方針

- **`湯 → よ`**: `温泉` との混同を防ぐため、エントリを `お湯（湯） → よ` と補足するか、会話例側で文脈を補う
- **`】【` ノイズ**: 会話例セクションの `（標準語訳）` 形式（`】【` ではなく `（）` の括弧）が影響しているか引き続き観察。フェーズ3実装後のチャット動作でも確認する
